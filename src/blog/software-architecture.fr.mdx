# Introduction

L'architecture logicielle n'est pas une question de diagrammes complexes, mais de **gestion de la complexité**.

## Les principes clés

1. Séparation des préoccupations
2. Testabilité
3. Indépendance du framework

> "Le code est lu bien plus souvent qu'il n'est écrit."
